# Feature Flags Tier Cascade Fix — Summary\n\n## Issue\n\n**Before:** Feature flags allowed independent per-plan settings without enforcing business logic.\n- Admin set \"Daily Edition\" FREE=ON, but left PLUS/PRO/VIP as NULL\n- VIP users (`anytime.sync@gmail.com`) saw \"Couldn't load\" errors\n- No automatic inheritance from lower tiers to higher tiers\n\n**Example from your screenshots:**\n```json\n{\n  \"feature_id\": \"ai_daily_edition\",\n  \"enabled_free\": true,\n  \"enabled_plus\": true,\n  \"enabled_pro\": null,      // Missing — should cascade from FREE\n  \"enabled_vip\": null       // Missing — should cascade from FREE\n}\n```\nResult: VIP users got errors even though FREE tier had access.\n\n## Solution\n\n**Tier Cascade Rule:** If a lower tier is granted access, all higher tiers automatically get access.\n\n### Tier Hierarchy\n```\nFREE (lowest) → PLUS → PRO → VIP (highest)\n```\n\n### Rule Logic\n- If **FREE is ON** → PLUS, PRO, VIP all ON (unless explicitly OFF)\n- If **PLUS is ON** (FREE off) → PRO, VIP all ON (unless explicitly OFF)\n- If **PRO is ON** (FREE/PLUS off) → VIP ON (unless explicitly OFF)\n- If **VIP is ON** only → only VIP has access\n\n### Result After Fix\nSame config now works correctly:\n```json\n{\n  \"feature_id\": \"ai_daily_edition\",\n  \"enabled_free\": true,\n  \"enabled_plus\": true,\n  \"enabled_pro\": null,\n  \"enabled_vip\": null\n}\n```\n✅ FREE ✅ PLUS ✅ PRO ✅ VIP (all have access — cascade applied)\n\n## Code Changes\n\nThree core files + tests + docs:\n\n### 1. `src/lib/feature-flags.ts` — Server-side gate\n\n**Added:** Cascade logic to `isPlanEnabledByOverride()`\n\n```typescript\nconst shouldCascade = (targetPlan: Plan): boolean => {\n  // If FREE is ON, cascade to PLUS/PRO/VIP\n  if (targetPlan === \"plus\" || targetPlan === \"pro\" || targetPlan === \"vip\") {\n    if (flag.enabled_free === true) return true;\n  }\n  // If PLUS is ON, cascade to PRO/VIP\n  if (targetPlan === \"pro\" || targetPlan === \"vip\") {\n    if (flag.enabled_plus === true) return true;\n  }\n  // If PRO is ON, cascade to VIP\n  if (targetPlan === \"vip\") {\n    if (flag.enabled_pro === true) return true;\n  }\n  return false;\n};\n```\n\n### 2. `src/app/api/admin/feature-flags/route.ts` — Admin API\n\n**Added:** `applyCascadeRule()` function to normalize saves\n\nWhen admin saves feature
